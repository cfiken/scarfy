1. **コードの品質**: 可読性、保守性、パフォーマンス
2. **ベストプラクティス**: 言語固有の推奨パターン
3. **セキュリティ**: 潜在的な脆弱性の確認
4. **テスト**: テストカバレッジや品質
5. **ドキュメント**: コメントやドキュメントの適切性
